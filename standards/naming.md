# DevForge Naming Standards

## Structure

- DevForge = product family
- RankForge = product
- Forge* = internal modules

## Examples

- ForgeCore
- ForgeQueue
- LinkForge
- SchemaForge
- VisionForge
- RankPulse

## Branch Naming

- feature/<name>
- fix/<name>
- chore/<name>

## File Naming

- lowercase-with-dashes for markdown and assets
- PascalCase for classes

## Principle

Names should be:
- descriptive
- consistent
- scalable across products
